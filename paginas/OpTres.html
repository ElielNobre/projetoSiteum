Desenvolvimento de Aplicação Web utilizando a plataforma JAVA EE e a utilização de UML para a documentação

RESUMO

O trabalho tenta simulação de um caso real de como se começa um projeto de uma aplicação Web proposto por um cliente, nesse caso um comercio eletrônico básico usando a plataforma Java Enterprise Edition. O trabalho começa no entendimento do cenário e dos requisitos do usuário que quer implementar aplicação especializada em venda de smartphones. Seguindo depois para os diagramas do projeto e documentação do sistema como o caso de uso, diagrama de classe e o de sequência e o esboço da interface front-end. Com a análise de pontos de função é feito uma estimativa dos os esforços e recursos requeridos no projeto. Por fim, na parte prática as páginas da aplicação foram codificadas com a interface de desenvolvimento Netbeans e o pacote Java JDK, o banco de dados Mysql e os drivers dos API’s previamente instalados. Criando as três tabelas do banco requerido pelo sistema, no caso a tabela cliente, pedido e produto, com auxílio da ferramenta Workbench. A arquitetura utilizada é o MVC com o framework JSF e as tecnologias de apoio CDI, JTA e EJB. Em resumo o usuário final vai ter acesso a uma página principal, página de descrição do produto, página de cadastro e de confirmação de compra. O trabalho finaliza com a descrição da experiencia com a plataforma bem como as vantagens e desvantagens de se usar a plataforma JAVA EE.

Palavras-chave: Aplicação Web, Java EE

  1. Introdução

A plataforma Java é um suporte de desenvolvimento que contempla uma linguagem de programação e uma máquina virtual, com ela é possível desenvolver estruturas voltadas para softwares e aplicações web ou mobile.

James Gosling, em 1991 propôs um sistema de comunicação entre dispositivos diferentes em uma casa, fazendo uma casa inteligente onde um controle central poderia interagir com equipamentos eletrodomésticos, é o que podemos considerar um princípio da internet das coisas, porém, fazendo uso de um dispositivo e não o protocolo TCP/IP. Os comandos de um dispositivo iriam executar em outro através de uma máquina virtual, esse princípio de execução multiplataforma permitiu e permite até hoje a comunicação entre sistemas de diferentes perfis e também a sua disseminação pela internet, sobretudo em aplicações web. Com o sucesso da plataforma, ela é hoje uma das mais completas e utilizadas no mundo que permite o desenvolvimento de aplicações para diversas finalidades. A versão corporativa da plataforma (Java Enterprise Edition) já inclui a plataforma padrão (Java Standard Edition) e permite fazer aplicações robustas, seguras e complexas além de oferecer uma ampla gama de API’s, segundo o tutorial do netbeans disponível no site netbeans.org, o Java EE contém um conjunto de tecnologias coordenadas que reduz significativamente o custo e a complexidade do desenvolvimento, implantação e gerenciamento de aplicações de várias camadas centradas no servidor.

Já tendo em mãos um computador devidamente preparado, com JDK, NetBeans e MySqlDB, um projeto de aplicação web pode ser devidamente produzido e testado antes de ser implementado definitivamente.

  1. Fundamentação Teórica

As tecnologias empregadas, primeiro no âmbito do planejamento: Análise de ponto de função. Uma vez levantados os requisitos antes do desenvolvimento do software e do banco de dados, como parte do planejamento, é possível através do APF estimar o tempo e os custos do desenvolvimento do projeto.

Essa técnica é focada no usuário final, aquele que usará o software para atender suas necessidades, e faz a contagem das funções, ou podemos dizer, das ações envolvidas no sistema. Esses pontos de função que são ações que podem ou não manipular e armazenar dados em um banco de dados, emitir uma forma de resposta, disparar uma forma de processamento, enfim, todos os pontos do processo que levem ao objetivo final.

As APF são de importância fundamental na elaboração dos diversos diagramas da documentação do projeto do software, pois como desenvolvedores devemos imaginar como os usuários vão interagir com o sistema e como será sua interface, antes de começar a codificação.

Tratando-se agora da codificação, as linguagens empregadas são: HTML, CSS, o XML e Java na plataforma, todos agregados na plataforma Java EE (entrerprise edition). O uso da mesma se justifica pelo JCP (java Comunity Process) que é a gerencia de padrões dessa tecnologia conhecidas como Java Specifications Request (JSR), a qual, segundo Marty Hall e Larry Brown, conta com o apoio de uma grande comunidade e com o interesse de empresas do setor.

A plataforma Java EE define um conjunto de especificações de frameworks para a facilitar o desenvolvimento para a web. A especificação para implementar o padrão MVC é o JSF (Java Server Faces) que possibilita desenvolver aplicações baseadas em componentes de forma ágil, abstraindo os detalhes do protocolo HTTP. comparativa Sbaraglini

O projeto vai utilizar a arquitetura MVC (Model, View, Controller), que nada mais é do que uma metodologia de se desenvolver uma aplicação de software. Segundo o site Devmedia, esse padrão traz como benefício isolar as regras do negócio, a lógica de apresentação e a interface com o usuário. A plataforma Java oferece todo o suporte ao MVC através do framework JSF (Java Server Faces). Traduzindo a sigla podemos chamar de Modelo, Interface e Controlador. Só para observar, o MVC está mais para um conceito, que visa organizar a arquitetura de um software em camadas, não há obrigatoriedade de desenvolve-lo com Java, por exemplo, pode ser usado o Python, que também seguem o paradigma da orientação a objetos.

O modelo vai cuidar das regras do negócio, mais precisamente a leitura e escrita dos dados no banco de dados. O controlador é a parte gerenciada pela plataforma, ou seja, vai gerir as páginas como se fossem objetos. A parte de interface é como o próprio nome sugere a parte frontal(front-end) escrita com HTML, XML e CSS, as vezes quando é necessário interatividade usa-se também o Javascript (linguagem de programação).

O uso do Java EE sem essa metodologia seria quase um desperdício visto que a intenção é desenvolver aplicações complexas e com camadas bem definidas facilitando dessa forma a manutenção, os programadores terão mais disponibilidade para se trabalhar os requisitos funcionais do sistema enquanto os não funcionais como segurança, escalabilidade, flexibilidade, etc já estarão implementados automaticamente. O MVC também é usado para fazer aplicativos móveis, diga-se de passagem.

Em resumo, os recursos utilizados foram:

  • Plataforma Java Enterprise Edition.
  • Arquitetura MVC;
  • IDE(Ambiente de desenvolvimento): NetBeans;
  • JPA (Java Persistence API), JSF (Java Server Face), JSP(Java Server Page), JTA (Java Transaction API) e EJB (Enterprise Java Beans);
  • Microsoft Visio para desenvolvimento do esboço da interface e dos Diagramas caso de uso, de sequência e diagrama de classe;
